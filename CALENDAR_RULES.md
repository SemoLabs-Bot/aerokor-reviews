# CALENDAR_RULES.md (Discord DM assistant)

목표: Google Calendar 조회/충돌 감지/일정 생성·수정은 **항상 승인 게이트**.

## 0) 기본 원칙

- 생성/수정/삭제는 승인 전까지 금지
- 충돌/과부하를 먼저 진단하고, 해결책 3안을 제시
- 시간대: Asia/Seoul (KST)

## 1) 기본 일정 규칙

- 기본 미팅 길이: 30분
- 60분 이상은 목적/아젠다가 명확할 때만
- 점심 블록: 12:00–13:00 (기본 보호)
- 이동/완충 버퍼:
  - 오프라인/외부 미팅: 앞뒤 15분
  - 온라인 미팅: 앞뒤 5분

## 2) 제목 규칙(일관성)

- 포맷: `[카테고리] 주제 — 상대/팀`
- 카테고리 예: `1:1`, `업무`, `채용`, `고객`, `집중`, `이동`
- 온라인 링크/회의실은 description에 넣고 제목에는 넣지 않음

## 3) 충돌 감지 규칙

- 겹침(overlap) 존재 시: 항상 경고
- 과부하(overload) 기준(다음 24h):
  - 미팅 총합 4시간 초과 또는
  - 연속 미팅 3개 이상 또는
  - 집중 블록(>=90분) 0개

## 4) 충돌 시 대안 3개 제시 규칙

충돌 발견 시, 아래 3안을 기본으로 제시:

1) **시간 이동(가장 가까운 빈 슬롯 3개)**
2) **길이 단축(예: 30→25 / 60→45)**
3) **비동기 전환(문서/메시지로 대체 + 체크포인트 시간만 잡기)**

## 5) 생성/수정 승인 템플릿(필수)

- 제안 일정 요약(날짜/시간/길이/참석자/장소)을 먼저 보여주고
- `이대로 생성/수정할까요? (예: "승인: 캘린더 생성")`로 승인 받기
