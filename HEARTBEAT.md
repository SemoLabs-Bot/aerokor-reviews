# HEARTBEAT.md
# 목적: 업무시간(KST)에만, DM 기반 비서로서 "새 이슈"만 짧게 알림.
# 출력 규칙: 이슈가 없으면 반드시 HEARTBEAT_OK 단 한 줄만 출력.

## 시간대/빈도(기본 가정)
# - KST 기준 평일 09:00~19:00 사이에만 실행되도록 gateway heartbeat 설정에서 activeHours 사용

## 1) Gmail triage (최근 60분 + 24시간)
- 최근 60분 동안 "unread" 신규 메일이 있으면:
  - (1) 긴급/답장필요/보관/항공 중 분류
  - (2) 각 카테고리별 카운트 + 상위 3건만 1~2줄 요약(민감정보 마스킹)
  - (3) 필요 시 "초안 작성" 제안(발송 금지)
- 지난 24시간 기준으로:
  - "긴급"이 남아있으면 최우선으로 알림

## 2) Calendar 충돌/과부하(다음 24시간)
- 겹치는 일정이 있으면 충돌 목록 + 해결 3안 제시
- 과부하(미팅 >4h, 연속 3개+, 집중블록 0개)면 알림 + 완화 3안

## 3) Flight check-in 감지/임박
- 지난 14일 메일/최근 일정 중 항공 키워드(itinerary, e-ticket, check-in 등) 감지 시:
  - 출발일/시간(가능하면)만 추출(예약번호/여권/전화 등 마스킹)
  - 체크인 오픈을 기본 24시간 전으로 계산(항공사 정보가 있으면 그 룰 우선)
  - 체크인 오픈 30분 전~오픈 시점 사이면 알림

## 4) Voice → Jira 후보 발생 여부(최근 60분)
- logs/jira-voice/runs/ 아래에 status=pending_candidates 또는 pending_approval 항목이 있으면:
  - run_id + 후보 개수 + 다음 액션(“/jira-from-transcript …” 또는 “승인: Jira 생성 …”)만 안내

## 출력 예외
- 위 1~4 중 아무 이슈도 없으면:
HEARTBEAT_OK
